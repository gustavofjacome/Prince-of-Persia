Sobre o projeto

Implementacao do classico Prince of Persia em linguagem Assembly MIPS, desenvolvida para o simulador MARS (MIPS Assembler and Runtime Simulator). O jogo conta com 2 fases, sistema de colisao por tiles, inimigo autonomo, combate com hitbox e efeitos sonoros via MIDI. Os sprites foram adaptados do jogo original de Super Nintendo e editados no Aseprite.

Como executar

  1. Baixe o MARS Simulator (Java Runtime necessario)
  2. Abra o arquivo main.asm (File → Open)
  3. Ative o Bitmap Display (Tools → Bitmap Display): 512x256, base 0x10010000, Unit 1x1, conecte ao MIPS
  4. Ative o Keyboard MMIO (Tools → Keyboard and Display MMIO Simulator), conecte ao MIPS
  5. Monte com F5 e execute (Run → Set Speed → maximum)
  6. Clique na janela do teclado MMIO para capturar as teclas

Arquitetura

O codigo segue uma arquitetura modular. Cada sistema do jogo esta em seu proprio arquivo .asm dentro de uma hierarquia de diretorios.

|
+-- util/controle/
|   +-- controlarCenarios.asm    # Loop principal (input, fisica, combate)

+-- util/renders/

Fluxo de execucao

O jogo executa em um loop infinito. Cada ciclo:

  1. Le o teclado via MMIO (endereco 0xFFFF0000)
  2. Interpreta a tecla e dispara a acao correspondente (pulo, ataque, movimento)
  3. Aplica fisica: gravidade (velocidade_y += 1), drift horizontal, colisao vertical, pouso
  4. Atualiza cooldown do ataque e verifica contato com o inimigo
  5. Verifica limites da tela e transicoes de cenario
  6. Redireciona para o renderizador correto (menu, fase 1 ou fase 2 + inimigo)

Sistema de colisao por tiles

O cenario e dividido em tiles de 16x16 pixels. Cada tile armazena um valor que determina seu comportamento de colisao:

ValorSignificadoDescricao
0VazioPersonagem pode passar
1SolidoParede, chao ou plataforma (bloqueia)
2PerigoAbismo, espinhos ou lava (morte instantanea)

A bounding box do personagem (9x42 pixels) e verificada em 6 pontos: os 4 cantos mais 2 pontos medios. Cada ponto e convertido em tile pela formula: 

tile_X = X / 16
tile_Y = Y / 16
indice = (tile_Y * 32 + tile_X) * 4    # 4 bytes por word

O maior valor entre os 6 pontos e retornado. Os mapas de colisao (stage1 e stage2) sao matrizes 32x16 armazenadas como words em tiles/stage1.asm e tiles/stage2.asm.

Variaveis globais: prince_x/y, inimigo_x/y, velocidade_y/x, no_chao, direcao, inimigo_vida, atacando, ataque_cooldown e outras.
controlesCenario
util/controle/controlarCenarios.asm (420 linhas)
Nucleo do loop de jogo. Faz polling do teclado via MMIO, mapeia teclas para acoes, executa fisica (gravidade, colisao, drift), gerencia combate e redireciona para o renderizador correto.
Teclas: W (pulo), Q/E (pulo diagonal), A/D (movimento), S (ataque), R (reset), X (sair)
iniciar_pulo / pulo_esquerda / pulo_direita
controlarCenarios.asm
Verifica se o personagem esta no chao, define velocidade vertical (-16) e horizontal (0, -5 ou +5), remove a flag de chao e reproduz som de pulo.
iniciar_ataque
controlarCenarios.asm
Verifica cooldown, define duracao do ataque (5 frames) e cooldown (10 frames), reproduz som de ataque e calcula hitbox. A hitbox se estende 65px para frente ou 56px para tras dependendo da direcao. Se acertar o inimigo, aplica dano (3 hits para matar).
aplicar_fisica
controlarCenarios.asm
Se no chao, verifica se o piso ainda esta presente. Se no ar, incrementa a gravidade (velocidade_y += 1). Calcula colisao vertical: se tile=0 permite movimento, tile=1 bloqueia/inverte, tile=2 mata. Trata pouso no chao (alinha Y ao tile) e drift horizontal do pulo diagonal.
morte_personagem
controlarCenarios.asm
Reproduz som de morte, reseta posicao do principe para (45, 54), zera fisica e estado de ataque, redireciona para o menu (cenario 0).
completou_fase / vai_para_cenario_2
controlarCenarios.asm
completou_fase: quando o principe atinge a borda direita no cenario 2, toca a fanfarra de vitoria e volta ao menu. vai_para_cenario_2: toca som de transicao e configura o cenario 2 com posicao inicial e vida do inimigo resetada.
existeCaracter / acionarCaracter
controlarCenarios.asm
Implementam o polling do teclado via MMIO. Leem o registrador Receiver Control em 0xFFFF0000 e retornam o codigo ASCII da tecla pressionada (ou 0 se nenhuma).
checar_colisao
util/controle/colisoes.asm (79 linhas)
Verifica colisao entre a bounding box 9x42 do personagem e o mapa de tiles. Testa 6 pontos no contorno do personagem. Para cada ponto, converte a coordenada (X,Y) para tile (divisao por 16) e consulta o mapa. Retorna o maior valor de tile encontrado (0=vazio, 1=solido, 2=morte).
Parametros: $a0 = X, $a1 = Y → Retorno: $v0 = tile (0/1/2)
obter_tile
colisoes.asm
Converte pixel (X,Y) em indice do mapa de tiles. Formula: indice = ((Y/16) * 32 + (X/16)) * 4. Seleciona o mapa correto baseado no cenario atual. Fora dos limites da tela, retorna tile solido (valor 1).

Parametros: $a0 = endereco do cenario, $a1 = largura, $a2 = altura

renderizarCenarioUm / renderizarCenarioDois / renderizarCenarioZero
renderizarCenario.asm
Renderizadores especificos de cada cenario:
- Cenario 1: se atualizar_fundo == 1, desenha o fundo completo. Senao, restaura so a area do sprite anterior (dirty rectangle). Seleciona o sprite correto (idle, pulo, ataque) conforme o estado do personagem.
- Cenario 2: mesma logica do cenario 1, mas adiciona o inimigo: restaura area antiga do inimigo, chama atualizar_inimigos e desenha o inimigo na nova posicao.
- Cenario 0: desenha o menu principal (imagem estatica).

Ferramentas de conversao

Dois scripts Python convertem imagens PNG para o formato Assembly usado pelo projeto:

conversor_cenarios.py
Converte imagens de cenarios para arrays de pixels em Assembly. Pixels com alpha 0 viram 0xFFFFFFFF (transparente no renderizador). Gera labels de largura e altura automaticamente.
conversor_sprites.py
Converte imagens de sprites para arrays de pixels. Pixels com alpha < 10 viram 0x00000000 (transparente no renderizador de sprites).
